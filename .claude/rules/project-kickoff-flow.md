# Project Kickoff Flow — Protocolo Padrão de Início de Projeto

## Regra Central

Todo projeto novo ou fase nova segue este fluxo de 11 etapas antes de qualquer linha de código ou copy final. Cada camada alimenta a próxima.

## Fluxo Canônico

```
CAMADA EXECUTIVA
1. @vision-chief (CEO)       → valida visão, direção e horizonte do produto
2. @cmo-architect (CMO)      → go-to-market, posicionamento, ICP, canais, CAC alvo

CAMADA DE MARCA E NARRATIVA
3. @brand-chief              → identidade, arquitetura de marca, diferenciação
4. @story-chief              → arco narrativo central (StoryBrand, Campbell, Duarte)

CAMADA DE OFERTA E COPY
5. @copy-chief               → copy de todas as peças (landing, email, social, ads)
6. @hormozi-chief            → oferta irresistível, precificação, hooks, retenção

CAMADA DE AQUISIÇÃO
7. @traffic-chief            → estratégia de tráfego pago (plataformas, criativos, escala)

CAMADA DE DADOS
8. @data-chief               → métricas de sucesso, CLV, activation rate, churn triggers

CAMADA TÉCNICA (paralela às camadas acima)
9. @cto-architect (CTO)      → decisões de stack, build vs buy, moat técnico
10. @dev @qa @architect      → implementação, testes, arquitetura

CAMADA DE CONSELHO (sob demanda — decisões de alto impacto)
11. @board-chair             → pivôs, fundraising, decisões existenciais
```

## Quando Usar o Fluxo Completo vs Parcial

| Situação | Fluxo |
|----------|-------|
| Novo produto do zero | Completo (etapas 1-10, 11 se necessário) |
| Nova fase de produto existente | A partir da etapa relevante |
| Novo canal de aquisição | Etapas 2, 5, 6, 7, 8 |
| Reposicionamento de marca | Etapas 1, 2, 3, 4, 5 |
| Nova feature técnica | Etapas 9, 10 |
| Decisão estratégica de alto risco | Etapa 11 (Advisory Board) |

## Distinção C-Level vs Advisory Board

| | C-Level Squad | Advisory Board |
|--|---------------|---------------|
| **Natureza** | Liderança executiva interna | Conselho externo estratégico |
| **Frequência** | Todo projeto/fase | Decisões de alto impacto |
| **Exemplos** | go-to-market, stack tech, posicionamento | fundraising, pivô de modelo, M&A |
| **Ativação** | Início do fluxo | Sob demanda |

## Proibições

- Não começar copy ou dev sem passar pelas etapas 1 e 2 (CEO + CMO)
- Não definir métricas de sucesso sem o data-chief (etapa 8)
- Não chamar Advisory Board como substituto do C-Level — são camadas distintas
- Não pular etapas "para economizar tempo" — cada camada alimenta a próxima
