# Story 1.2: Claude API + SQLite + Fluxo de Conversação Base

## Status

Ready for Review

## Story

**As a** desenvolvedor do EasySite,
**I want** o bot Leo integrado com a Claude API e com banco de dados SQLite para persistir conversas,
**so that** o bot consiga manter contexto entre mensagens e responder de forma inteligente usando o system prompt do Leo.

## Acceptance Criteria

1. `claude_client.py` implementado com função `get_response(phone, message)` que retorna string
2. System prompt do Leo carregado a partir de `docs/easysite/bot/system-prompt.md`
3. Banco SQLite criado automaticamente ao iniciar o servidor (`db/easysite.db`)
4. Tabela `leads` criada com campos: id, phone, name, source, status, last_bot_message_at, created_at, updated_at
5. Tabela `conversations` criada com campos: id, lead_id, role, content, timestamp
6. A cada mensagem recebida: histórico da conversa é buscado do banco e enviado para Claude junto com a nova mensagem
7. Resposta do Claude é salva no banco e enviada ao cliente via Evolution API
8. Novo lead é criado automaticamente no banco se phone não existir (source=direct_ad por padrão)
9. Teste de integração: enviar mensagem de teste → bot responde via WhatsApp com resposta do Leo
10. Servidor mantém estabilidade após 10 mensagens sequenciais no mesmo número

## Scope

**IN:**
- Integração Claude API (claude-haiku-4-5)

### Integração Claude API

Usar SDK oficial Anthropic. Adicionar `anthropic` ao requirements.txt.

```python
import anthropic
from pathlib import Path

client = anthropic.Anthropic(api_key=os.getenv("CLAUDE_API_KEY"))

SYSTEM_PROMPT = Path("system-prompt.md").read_text(encoding="utf-8")

def get_response(conversation_history: list[dict]) -> str:
    response = client.messages.create(
        model="claude-haiku-4-5-20251001",
        max_tokens=1024,
        system=SYSTEM_PROMPT,
        messages=conversation_history
    )
    return response.content[0].text
```

### SQLite — criação das tabelas (db/models.py)

```python
import sqlite3, os

DB_PATH = os.path.join(os.path.dirname(__file__), "easysite.db")

def init_db():
    conn = sqlite3.connect(DB_PATH)
    conn.execute("""
        CREATE TABLE IF NOT EXISTS leads (
            id INTEGER PRIMARY KEY AUTOINCREMENT,
            phone TEXT UNIQUE NOT NULL,
            name TEXT,
            source TEXT DEFAULT 'direct_ad',
            status TEXT DEFAULT 'new',
            last_bot_message_at TEXT,
            created_at TEXT DEFAULT (datetime('now')),
            updated_at TEXT DEFAULT (datetime('now'))
        )
    """)
    conn.execute("""
        CREATE TABLE IF NOT EXISTS conversations (
            id INTEGER PRIMARY KEY AUTOINCREMENT,
            lead_id INTEGER NOT NULL,
            role TEXT NOT NULL,
            content TEXT NOT NULL,
            timestamp TEXT DEFAULT (datetime('now')),
            FOREIGN KEY (lead_id) REFERENCES leads(id)
        )
    """)
    conn.commit()
    conn.close()
```

### Fluxo completo por mensagem recebida (main.py atualizado)

```
1. Recebe webhook
2. Extrai phone, message, name
3. Ignora fromMe=true
4. Busca ou cria lead no banco
5. Salva mensagem do usuário em conversations (role=user)
6. Busca histórico completo da conversa desse lead
7. Chama claude_client.get_response(historico)
8. Salva resposta do Claude em conversations (role=assistant)
9. Atualiza last_bot_message_at no lead
10. Envia resposta via whatsapp_client.send_message()
```

### Limite de histórico

Enviar no máximo as últimas 20 mensagens para evitar context overflow:
```python
history = get_conversation_history(lead_id)[-20:]
```

curl -X POST http://localhost:5000/webhook \
  -H "Content-Type: application/json" \
  -d '{
    "event": "messages.upsert",
    "instance": "easysite",
    "data": {
      "key": {"remoteJid": "5511999999999@s.whatsapp.net", "fromMe": false, "id": "TEST002"},
