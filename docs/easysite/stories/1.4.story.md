# Story 1.4: Follow-up Scheduler (APScheduler)

## Status

Ready for Review

## Story

**As a** desenvolvedor do EasySite,
**I want** o bot Leo enviar follow-ups automáticos para leads que pararam de responder,
**so that** nenhum lead quente seja perdido por falta de acompanhamento, seguindo a régua de 1h, 24h, 72h e 7 dias.

## Acceptance Criteria

1. Tabela `follow_up_queue` criada com campos: id, lead_id, scheduled_at, message_type, status, sent_at
2. APScheduler integrado ao FastAPI, rodando em background, verificando fila a cada 5 minutos
3. Quando lead entra em status qualifying ou proposal_sent, agendamento de follow-up é criado automaticamente
4. Follow-up de 1h é cancelado se lead responder antes
5. Follow-up de 24h é cancelado se lead responder antes
6. Follow-up de 72h é cancelado se lead responder antes
7. Após envio do follow-up de 7 dias, status do lead muda para closed_lost
8. Follow-ups pendentes sobrevivem a restart do servidor (persistidos no banco)
9. Log claro no console para cada follow-up enviado: `[FOLLOWUP] {phone} - {message_type}`
10. Teste: criar lead, simular inatividade, verificar que follow-up de 1h é enviado

## Scope

**IN:**
- Tabela follow_up_queue no SQLite
- APScheduler integrado ao FastAPI
- Agendamento automático ao atualizar status do lead
- Cancelamento ao receber nova mensagem do lead
- 4 mensagens de follow-up conforme system prompt

**OUT:**
- Fluxo de PIX (Story 1.5)
- Notificações admin (Story 1.5)

### Inicialização do APScheduler (main.py)

```python
from apscheduler.schedulers.asyncio import AsyncIOScheduler

scheduler = AsyncIOScheduler()

@app.on_event("startup")
async def startup_event():
    init_db()
    scheduler.add_job(process_follow_ups, 'interval', minutes=5, id='follow_up_job')
    scheduler.start()

@app.on_event("shutdown")
async def shutdown_event():
    scheduler.shutdown()
```

### Lógica de agendamento

Ao atualizar status do lead para qualifying ou proposal_sent:

```python
def schedule_follow_ups(lead_id: int, from_now: datetime):
    intervals = {
        "1h":  timedelta(hours=1),
        "24h": timedelta(hours=24),
        "72h": timedelta(hours=72),
        "7d":  timedelta(days=7)
    }
    for message_type, delta in intervals.items():
        scheduled_at = (from_now + delta).isoformat()
        insert_follow_up(lead_id, scheduled_at, message_type)
```

### Cancelar follow-ups ao receber mensagem

Ao processar qualquer mensagem do cliente:
```python
cancel_pending_follow_ups(lead_id)
schedule_follow_ups(lead_id, datetime.now())  # reagendar do zero
```

### Job de processamento (scheduler/follow_up.py)

```python
def process_follow_ups():
    now = datetime.now().isoformat()
    pending = get_pending_follow_ups(now)  # WHERE scheduled_at <= now AND status = 'pending'
    for item in pending:
        lead = get_lead_by_id(item["lead_id"])
        name = lead["name"] or "você"
        message = FOLLOW_UP_MESSAGES[item["message_type"]].format(name=name)
        send_message(lead["phone"], message)
        mark_follow_up_sent(item["id"])
        print(f"[FOLLOWUP] {lead['phone']} - {item['message_type']}")
        if item["message_type"] == "7d":
            update_lead_status(lead["id"], "closed_lost")
```
