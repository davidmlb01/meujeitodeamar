# Story 1.5: Fluxo PIX + Notificações Admin para David

## Status

Ready for Review

## Story

**As a** desenvolvedor do EasySite,
**I want** o bot Leo detectar comprovantes de PIX, notificar David via WhatsApp e processar a confirmação dele para enviar o formulário de briefing ao cliente,
**so that** o fluxo de pagamento seja automatizado sem intervenção manual além da confirmação do David.

## Acceptance Criteria

1. Tabela `escalations` criada com campos: id, lead_id, reason, pix_code, notified_at, resolved_at
2. Bot detecta quando cliente envia mensagem com padrões de comprovante PIX (palavras-chave + análise Claude)
3. Ao detectar PIX: bot responde ao cliente com mensagem padrão de aguardo
4. Ao detectar PIX: status do lead muda para pix_pending
5. Ao detectar PIX: Leo envia mensagem para DAVID_PHONE com nome, telefone e código de confirmação (ex: "SIM-042")
6. Ao receber mensagem de DAVID_PHONE com "SIM-{código}": bot localiza o lead correto e muda status para pix_confirmed
7. Após confirmação: bot envia link do formulário de briefing para o cliente (`{{LINK_FORMULARIO}}` substituído pelo valor real em .env)
8. Mensagens de DAVID_PHONE que não são comandos admin são ignoradas (não tratadas como lead)
9. Tabela `escalations` registra cada PIX detectado com timestamp e código
10. Teste: simular comprovante PIX → verificar notificação David → simular resposta David → verificar envio do formulário

## Scope

**IN:**
- Detecção de comprovante PIX na conversa
- Notificação WhatsApp para David com código de confirmação
- Processamento de resposta admin do David
- Envio automático do formulário de briefing após confirmação
- Tabela escalations para auditoria

**OUT:**
- Sistema de escalação para outros tipos de ocorrência (pode ser expandido futuramente)
- Confirmação de pagamento via API do banco (fora do escopo, confirmação é manual pelo David)
- Integração com Kiwify ou outro processador de pagamento

### Detecção de PIX

Dois níveis de detecção:

**Nível 1 — palavras-chave (rápido, sem custo de API):**
```python
PIX_KEYWORDS = [
    "comprovante", "pix", "transferência", "pagamento",
    "paguei", "transferi", "fiz o pix", "mandei o pix",
    "recibo", "pagamento realizado"
]

def has_pix_keywords(message: str) -> bool:
    message_lower = message.lower()
    return any(kw in message_lower for kw in PIX_KEYWORDS)
```

**Nível 2 — Claude confirma (se keywords detectadas):**
Adicionar ao system prompt de detecção:
```
O cliente enviou a mensagem abaixo. Responda apenas "SIM" se parece ser um comprovante ou confirmação de pagamento PIX, ou "NAO" caso contrário.
Mensagem: "{message}"
```

Usar claude-haiku para essa chamada de classificação (barata e rápida).

### Geração do código de confirmação PIX

```python
import random, string

def generate_pix_code() -> str:
    return ''.join(random.choices(string.digits, k=3))  # ex: "042"
```

### Fluxo completo ao detectar PIX

```
1. Detectar PIX (keywords + Claude classifica)
2. Gerar código único (ex: "042")
3. Salvar em escalations (lead_id, reason=pix_received, pix_code=042)
4. Atualizar status do lead para pix_pending
5. Cancelar follow-ups pendentes
6. Responder ao cliente:
   "Recebi seu comprovante, [Nome]! Vou verificar o pagamento e te retorno em seguida.
   Nosso horário de verificação é de segunda a sexta, das 9h às 18h..."
7. Notificar David:
   "PIX recebido de [Nome] ([telefone]).
   Responda SIM-042 para confirmar e enviar o formulário."
```

### Processamento de mensagem do David

```python
DAVID_PHONE = os.getenv("DAVID_PHONE")

def handle_admin_message(phone: str, message: str):
    if phone != DAVID_PHONE:
        return  # não é David

    message = message.strip().upper()

    if message.startswith("SIM-"):
        code = message.replace("SIM-", "")
        confirm_pix(code)
    # Outros comandos admin podem ser adicionados aqui
```

### Confirmação do PIX

```python
def confirm_pix(pix_code: str):
    escalation = get_escalation_by_code(pix_code)
    if not escalation or escalation["resolved_at"]:
        send_message(DAVID_PHONE, f"Código {pix_code} não encontrado ou já processado.")
        return

    lead = get_lead_by_id(escalation["lead_id"])
    update_lead_status(lead["id"], "pix_confirmed")
    mark_escalation_resolved(escalation["id"])

    briefing_url = os.getenv("BRIEFING_FORM_URL", "{{LINK_FORMULARIO}}")
    send_message(lead["phone"],
        f"Pagamento confirmado, {lead['name'] or ''}! Agora é só preencher esse formulário rapidinho (leva uns 5 minutos):\n\n{briefing_url}\n\nAssim que receber suas respostas, a gente já começa a criar o seu site. Entrega em até 24 horas.")

    send_message(DAVID_PHONE, f"PIX-{pix_code} confirmado. Formulário enviado para {lead['phone']}.")
```
