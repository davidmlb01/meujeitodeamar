# Story 1.1: Infraestrutura Base — Projeto Python + FastAPI + Evolution API

## Status

Ready for Review

## Story

**As a** desenvolvedor do EasySite,
**I want** ter a infraestrutura base do bot configurada e rodando na VPS,
**so that** as próximas stories possam construir os fluxos de conversação sobre uma base técnica sólida, com webhook ativo e Evolution API recebendo mensagens.

## Acceptance Criteria

1. Estrutura de pastas do projeto criada em `docs/easysite/bot/`
2. `requirements.txt` com todas as dependências (fastapi, uvicorn, httpx, python-dotenv, requests)
3. `.env.example` documentado com todas as variáveis necessárias (sem valores reais)
4. `main.py` com servidor FastAPI na porta 5000, endpoint `/webhook` respondendo POST
5. `main.py` loga no console toda mensagem recebida via webhook (phone + message body)
6. `whatsapp_client.py` com função `send_message(phone, text)` que envia mensagem via Evolution API
7. Teste de smoke: ao enviar POST manual para `/webhook` com payload de teste, servidor responde 200 e loga a mensagem
8. `ecosystem.config.js` criado para PM2 apontar para `main.py`
9. `README.md` na pasta `bot/` com instruções de deploy na VPS (step-by-step)
10. Todo o código sobe sem erros: `uvicorn main:app --port 5000`

### Variáveis de ambiente (.env.example)

```
# Evolution API
EVOLUTION_API_URL=http://localhost:8080
EVOLUTION_API_KEY=sua_chave_aqui
EVOLUTION_INSTANCE=easysite

# Claude API
CLAUDE_API_KEY=sua_chave_aqui

# Números WhatsApp
BOT_PHONE=55119XXXXXXXX        ← número do bot (sem +, sem espaço)
DAVID_PHONE=55119XXXXXXXX      ← número do David para notificações admin

# Config
PORT=5000
```

### Payload Evolution API (webhook recebe)

```json
{
  "event": "messages.upsert",
  "instance": "easysite",
  "data": {
    "key": {
      "remoteJid": "5511999999999@s.whatsapp.net",
      "fromMe": false,
      "id": "ABCDEF123"
    },
    "message": {
      "conversation": "Olá, quanto custa o site?"
    },
    "pushName": "João Silva"
  }
}
```

### Extração do payload

```python
phone = data["data"]["key"]["remoteJid"].replace("@s.whatsapp.net", "")
message = data["data"]["message"].get("conversation", "")
name = data["data"].get("pushName", "")
from_me = data["data"]["key"]["fromMe"]
```

### Ignorar mensagens do próprio bot

```python
if from_me:
    return {"status": "ignored"}
```

### Envio via Evolution API

```
POST {EVOLUTION_API_URL}/message/sendText/{EVOLUTION_INSTANCE}
Headers: apikey: {EVOLUTION_API_KEY}
Body: { "number": "5511999999999", "text": "Olá!" }
```

### PM2 (ecosystem.config.js)

```javascript
module.exports = {

    env: {
      PYTHONUNBUFFERED: "1"
    }
  }]
}
```

## Testing

### Smoke test manual (sem Evolution API)

```bash
# Subir servidor localmente
cd docs/easysite/bot
pip install -r requirements.txt
uvicorn main:app --port 5000

# Em outro terminal, enviar payload de teste
curl -X POST http://localhost:5000/webhook \
  -H "Content-Type: application/json" \
  -d '{
    "event": "messages.upsert",
    "instance": "easysite",
    "data": {
      "key": {"remoteJid": "5511999999999@s.whatsapp.net", "fromMe": false, "id": "TEST001"},
      "message": {"conversation": "Olá, quanto custa?"},
      "pushName": "Teste"
    }
  }'

# Esperado: {"status": "ok"} + log no console mostrando phone e mensagem
```

### Verificações obrigatórias

- [ ] `uvicorn main:app --port 5000` sobe sem erros
- [ ] POST para `/webhook` retorna 200
- [ ] Console loga phone e mensagem recebida
- [ ] Mensagens com `fromMe: true` são ignoradas (retornam `{"status": "ignored"}`)
- [ ] `send_message()` em `whatsapp_client.py` não lança exceção com dados válidos
