# Story 1.3: Roteamento das 3 Jornadas (A, B, C)

## Status

Ready for Review

## Story

**As a** desenvolvedor do EasySite,
**I want** o bot Leo identificar e rotear leads pelas 3 jornadas corretamente,
**so that** cada lead receba a abordagem certa com base na sua origem (anúncio, landing page ou prospecção ativa).

## Acceptance Criteria

1. Tabela `leads` atualizada com campos: company, ramo, cidade (para Jornada C)
2. Endpoint `POST /api/lead/landing` recebe dados da Landing Page (name, phone, ramo) e cria lead com source=landing_page
3. Endpoint `POST /api/lead/prospecting` recebe dados da planilha (company, phone, ramo, cidade) e cria lead com source=prospecting
4. Router identifica jornada do lead pelo campo `source` e ajusta o system prompt de contexto
5. Jornada A (direct_ad): bot faz saudação + coleta básica (ramo, logo, domínio) sem revelar preço imediato
6. Jornada B (landing_page): bot inicia contato proativo com nome e nicho já conhecidos
7. Jornada C (prospecting): bot inicia contato proativo com nome da empresa, ramo e cidade já conhecidos
8. Endpoint `POST /api/lead/prospecting/batch` aceita lista de leads da planilha e os enfileira
9. Leads de Jornada B e C recebem mensagem inicial automaticamente ao serem criados
10. Teste: criar lead via cada endpoint e verificar mensagem inicial correta enviada via WhatsApp

## Scope

**IN:**
- Endpoints para criação de leads externos (Landing Page, Prospecção)
- Lógica de roteamento por source
- Contexto adicional no prompt por jornada
- Disparo automático da mensagem inicial para Jornadas B e C

**OUT:**
- Follow-up scheduler (Story 1.4)
- Fluxo de PIX (Story 1.5)
- Integração direta com Google Sheets API (pode ser via script externo que chama o endpoint)

### Contexto adicional por jornada (injetado antes do histórico)

O router injeta um system message adicional de contexto baseado na jornada:

**Jornada A (direct_ad):**
```
Contexto: Lead chegou de anúncio no Meta. Provavelmente perguntou "quanto custa?" ou "como funciona?".
NÃO revelar preço imediatamente. Seguir Jornada A do system prompt.
```

**Jornada B (landing_page):**
```
Contexto: Lead veio da Landing Page. Já preencheu formulário.
Nome: {name} | Ramo: {ramo}
Este é o PRIMEIRO contato. Seguir Jornada B do system prompt — contato proativo já com dados do lead.
```

**Jornada C (prospecting):**
```
Contexto: Lead da prospecção ativa (Google Maps).
Empresa: {company} | Ramo: {ramo} | Cidade: {cidade}
Este é o PRIMEIRO contato. Seguir Jornada C do system prompt — abordagem de prospecção.
```

### Disparo automático (Jornadas B e C)

Ao criar lead via `/api/lead/landing` ou `/api/lead/prospecting`:
1. Criar lead no banco
2. Montar contexto da jornada
3. Chamar Claude com histórico vazio + contexto da jornada
4. Salvar resposta em conversations (role=assistant)
5. Enviar via Evolution API

### Endpoints

```python
POST /api/lead/landing
Body: {"name": "João", "phone": "5511999999999", "ramo": "salão de beleza"}

POST /api/lead/prospecting
Body: {"company": "Salão X", "phone": "5511999999999", "ramo": "salão", "cidade": "São Paulo"}

POST /api/lead/prospecting/batch
Body: [{"company": "...", "phone": "...", "ramo": "...", "cidade": "..."}, ...]
```

### Integração Google Sheets (Jornada C)

O script externo (Python) que lê a planilha chama `POST /api/lead/prospecting/batch`.
O bot não lê a planilha diretamente — responsabilidade do script separado.

O script externo pode rodar:
- Na própria VPS via cron job (`0 9 * * * python3 /root/sheets_sync.py`)
- Ou localmente quando David quiser disparar nova rodada
