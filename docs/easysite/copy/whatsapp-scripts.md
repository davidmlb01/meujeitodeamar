## 7. Notas de Configuração

### Variáveis para substituir

| Variável | Descrição | Exemplo |
|----------|-----------|---------|
| `{nome}` | Nome do contato ou do negócio | "João" / "Barbearia do João" |
| `{ramo}` | Segmento do negócio | "barbearia" / "eletricista" / "dentista" |
| `{cidade}` | Cidade de atuação | "Campinas" / "Recife" |
| `{n_avaliacoes}` | Número de avaliações no Google | "47" / "12" |
| `{nome_quem_indicou}` | Cliente que fez a indicação | "Carlos" |
| `{nome_indicado}` | Pessoa indicada | "Marcos" |

### Regras de rotação

- Seleção aleatória entre as 6 variações
- NUNCA enviar a mesma variação duas vezes para o mesmo DDD no mesmo dia
- Variação E só usar quando `{n_avaliacoes}` >= 5 (senão soa estranho)
- Variação F funciona melhor para ramos competitivos (barbearia, dentista, restaurante)
- Variação D funciona melhor para serviços (eletricista, encanador, personal trainer)

### Limites operacionais

- Limite diário: 25 mensagens outbound por número
- Horário: 9h às 17h (dias úteis)
- Intervalo mínimo entre mensagens: 3 a 5 minutos
- Follow-up automático: dia 1, dia 3, dia 7
- Stop automático: se lead responder qualquer coisa, parar sequência e notificar operador
