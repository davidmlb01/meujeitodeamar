# Post Quality Checklist

Executar antes de entregar qualquer post ao usuário. Se qualquer item falhar, corrigir antes de entregar.

## Hook (primeira linha)
- [ ] A primeira linha funciona sozinha como uma sentença completa
- [ ] Para o scroll — não é genérica, não é óbvia
- [ ] Não começa com "Eu", "Nós", "A Destaka" ou nome da marca
- [ ] Não começa com pergunta retórica fraca ("Você sabia que...?")

## Corpo
- [ ] Uma ideia só — não tenta dizer duas coisas
- [ ] Parágrafos com no máximo 3 linhas
- [ ] Nenhuma linha com mais de 20 palavras (regra geral)
- [ ] Sem palavras proibidas: otimizar, otimização, algoritmo, SEO, expertise, soluções, ecossistema, jornada, inovação
- [ ] Usa vocabulário do ICP: consultório (não empresa), pacientes (não clientes), aparecer (não ranquear)
- [ ] Sem linguagem corporativa
- [ ] Sem filler: "basicamente", "na verdade", "de fato", "certamente", "obviamente"

## Prova e Credibilidade
- [ ] Sem prova social falsa (números inventados, depoimentos fictícios)
- [ ] Sem preço do produto mencionado
- [ ] Dados específicos têm fonte implícita ou explícita

## CTA (última linha ou bloco)
- [ ] CTA é específico — não é "o que você acha?" ou "deixe seu comentário"
- [ ] CTA é uma pergunta, ação ou reflexão direcionada
- [ ] Não termina vendendo o produto diretamente

## Formato LinkedIn
- [ ] Sem markdown (negrito/itálico não renderizam)
- [ ] Linhas em branco entre parágrafos
- [ ] Máximo 3 hashtags, no final (ou nenhum)
- [ ] Dentro do limite de palavras definido no brief (100-350 palavras)

## Qualidade Geral
- [ ] Lê naturalmente em português brasileiro — não parece traduzido
- [ ] Soa como a marca, não como o agente que escreveu
- [ ] Se removida a primeira linha, o post ainda faz sentido
- [ ] Se removido o CTA, o post ainda entrega valor
